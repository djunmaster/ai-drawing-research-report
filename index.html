DWG → DXF 转换

选型规则匹配

Excel BOM 输出
用 openpyxl 生成格式化Excel，含产品名称/型号/数量/单位/所属柜体，兼容现有BOM模板格式。

ezdxf 文字提取核心代码
⚠️
重要：设备标注可能存在于3种不同位置，必须全部提取，遗漏任何一种都会导致信息不完整。
📌 TEXT / MTEXT 实体

图纸中的文字标注，直接附着在图上。MTEXT 用 .plain_text()，会自动去除格式代码。

🧩 INSERT.attribs 属性

设备图块（断路器/CT符号）上的属性标注。很多设备参数存在这里，必须遍历 INSERT 实体。

📦 嵌套块（递归展开）

块内嵌套块的文字。用 disassemble.recursive_decompose 展开，确保不遗漏。

python
from ezdxf import disassemble, query as dxf_query
from collections import defaultdict

def extract_all_texts(doc):
    msp = doc.modelspace()
    layer_texts = defaultdict(list)

    # 1. 直接文字实体
    for entity in msp.query("TEXT MTEXT"):
        txt = (entity.plain_text() if entity.dxftype() == "MTEXT"
               else entity.dxf.text)
        if txt and txt.strip():
            layer_texts[entity.dxf.layer].append(txt.strip())

    # 2. 图块属性（关键！设备参数在这里）
    for insert in msp.query("INSERT"):
        for attrib in insert.attribs:
            val = attrib.dxf.text
            if val and val.strip():
                layer_texts[insert.dxf.layer].append(
                    f"{attrib.dxf.tag}={val.strip()}")

    # 3. 递归展开嵌套块
    all_nested = dxf_query.EntityQuery(
        disassemble.recursive_decompose(msp))
    for e in all_nested.query("TEXT MTEXT"):
        txt = e.plain_text() if e.dxftype() == "MTEXT" else e.dxf.text
        if txt and txt.strip():
            layer_texts[f"nested:{e.dxf.layer}"].append(txt.strip())

    return layer_texts
🤖
AI 结构化提取（Claude + instructor）
为什么选 instructor？
  • LLM 偶尔输出格式不对 → 自动重试（max_retries=3）
  • Pydantic 字段定义即文档，自动校验数据类型
  • 与 Anthropic SDK 无缝集成，一行代码切换
  • 支持提示缓存（大图纸成本可降低 90%）
安装命令
pip install "instructor[anthropic]"

已有 anthropic SDK v0.76.0，装 instructor 后即可使用。无需额外配置。

python
import anthropic, instructor
from pydantic import BaseModel, Field
from typing import List, Optional

# 数据模型定义（需求一：列头柜）
class OutgoingCircuit(BaseModel):
    circuit_label:    str            = Field(description="出线回路标签")
    breaker_rating_A: Optional[int]  = Field(None, description="断路器额定电流(A)")
    ct_model:         Optional[str]  = Field(None, description="CT型号,如LDCT-400X1")
    cable_spec:       Optional[str]  = Field(None, description="电缆规格")

class ColumnHeadCabinet(BaseModel):
    cabinet_id:        str                    = Field(description="柜体编号")
    incoming_circuits: List[OutgoingCircuit]  = Field(default_factory=list)
    outgoing_circuits: List[OutgoingCircuit]  = Field(default_factory=list)

class ExtractionResult(BaseModel):
    column_head_cabinets: List[ColumnHeadCabinet]

# 初始化客户端
client = instructor.from_anthropic(
    create=anthropic.Anthropic(),
    mode=instructor.Mode.TOOLS
)

# 结构化提取（自动Pydantic校验+重试）
result: ExtractionResult = client.create(
    model="claude-sonnet-4-6",
    max_tokens=4096,
    system="你是数据中心电气工程专家，从配电系统图DXF文字中提取列头柜配置信息。",
    messages=[{"role": "user", "content": context_text}],
    response_model=ExtractionResult,
    max_retries=3   # 格式错误自动重试
)

电气规则库（需求二核心）
CT-断路器匹配合理性判断（GB/T 1208-2006 · 70%原则）
核心公式：CT一次额定电流 = 线路最大电流 ÷ 0.7，向上取GB标准值
断路器额定电流 / CT一次额定电流 的比值判断区间：
CT偏大
✓ 合理区间
CT偏小
0%30%60%120%150%200%+
✅
60%～120% 正常合理范围
⚠️
120%～150% CT轻度偏小，警告
🚨
>150% CT严重偏小，铁芯饱和
常用CT变比选型对照表
断路器额定（A）推荐CT变比工作比例典型应用
100A100/5100%（上限）小容量出线
160A200/580%（最佳）中小容量出线
250A300/583%（良好）中等容量出线
400A500/580%（最佳）较大容量进线
630A600/5 或 800/578%～105%大容量进线
1000A1000/5100%（上限）主配电进线
电缆-断路器匹配规则（GB 50054-2011）
🚨 电缆过细（危险）
电缆载流量 IZ < 断路器额定电流 In
→ 过载保护失效，电缆过热，火灾风险
⚠️ 电缆过粗（浪费）
电缆载流量 IZ > 断路器额定电流 × 3
→ 无安全风险，但成本浪费，设计失误
铜芯电缆载流量速查（穿管，25°C）： 16mm²→64A · 35mm²→104A · 50mm²→128A · 95mm²→192A · 185mm²→296A · 240mm²→352A

⚠️
风险与注意事项
🚨
图层名不统一（最大风险）
不同设计院的图层命名完全不同，无法写死代码。解决方案：全量提取所有文字，不依赖图层过滤；先探索再编码。
⚠️
SHX中文字体乱码
老图纸（R2004及以前，GBK编码）可能出现乱码。解决：ezdxf.readfile("x.dxf", encoding="cp936")
⚠️
设备参数在ATTRIB里，不在TEXT里
断路器/CT符号的型号参数通常存在图块属性ATTRIB中，必须遍历 INSERT.attribs，仅查 TEXT 实体会漏掉大量信息。
ℹ️
选型规则文件格式需确认
需求一的规则匹配依赖选型规则Excel的格式（列名、电流档位定义等）。需要与刘琦确认标准格式后才能完成这部分开发。
ℹ️
备用方案（无法安装ODA时）
经实证验证：DWG二进制中文字以GBK明文存储，可直接字节搜索提取。但这只能得到文字，无图层/坐标/块关系信息，精度较低。
