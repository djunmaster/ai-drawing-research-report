架构关键决策

  • 三层分离：预处理(纯数据) / AI 识别(感知) / 业务逻辑(决策)，各层可独立替换升级。
  • JSON Schema 中心化：设备对象库 D1 是所有需求共享的「事实单一源」，避免重复识别 [DRY]。
  • 规则强校验：选型 + 电气匹配都走 json-rules-engine + DMN，LLM 只做提参/解释，不允许直接出选型结论。
  • 国产化优先：YOLOv11-OBB / PP-OCRv6 / Qwen2.5-VL / AntV G6 全部国产合规。

4 类核心校验（基于 GB 50054 / DL/T 5222 / IEC 61850）

校验项规则引用

电缆汇总与选型载流量 I_z ≥ I_n（温度/海拔/谐波降容）；压降 ΔU ≤ 5%；短路耐受 I²t 达标DL/T 5222 附表 D.11
CT-电缆匹配CT 一次额定电流 ≥ 回路工作电流；精度满足计量/保护；二次负荷 ≤ 额定负荷IEC 61850-9-2

一次识别，多处使用的对象流（核心 Pydantic 模型）

# 共享层: 设备对象库
class Cabinet(BaseModel):
    id: str                       # QF1, 列头柜-A1
    type: Literal["列头柜","补偿柜","进线柜","馈线柜"]
    capacity_kva: float | None
    voltage_kv: float
    circuits: list["Circuit"]
    main_breaker: "Breaker" | None
    ct_devices: list["CT"]
    bbox: tuple[int,int,int,int] | None  # CAD 坐标

class CT(BaseModel):
    id: str; ratio: str           # 600/5
    accuracy_class: str           # 0.5/0.5/10P10
    burden_va: float; primary_a: float; secondary_a: float = 5
    purpose: Literal["计量","保护","测量"]

class Breaker(BaseModel):
    id: str; model: str           # VD4-12, MTZ2 10N1
    rated_a: float; breaking_ka: float; peak_ka: float; poles: int
  • 需求二（CT-电缆-断路器匹配） = 对设备对象库的关系运算，无需重新识别。
  • 需求三（组态图） = 对设备对象库的拓扑图可视化。
  • 需求一（选型） = 对设备对象的规则匹配 + RAG 召回。

9核心结论与推荐

「CAD 解析一次，设备对象复用三次，选型/校验/组态图并行输出」—— 三个需求是同一技术栈的三个输出面，而非三个独立产品，DRY 是商业可行性的关键。

P0 核心推荐

  • P0 共享 CAD 解析服务（DWG→DXF→ezdxf→设备 JSON）—— 三个需求共用，DRY 核心。
  • P0 Pydantic v2 设备对象库 + 统一 ScadaTopology Schema —— 一次识别，多处使用。
  • P0 json-rules-engine + DMN 决策表 —— 选型+校验共用，LLM 只做提参/解释。
  • P0 BGE-M3 + BGE-Reranker + pgvector —— 国产 RAG 全栈。
  • P1 AntV G6 v5 + Mermaid + mingrammer/diagrams —— 三端组态图渲染。
  • P1 openpyxl + Jinja2 + Pandera —— Excel 输出 + 双层校验。
  • P1 6 个月 MVP，200–500 万投入 —— 商业化路径清晰。
